Applications for Handhelds:

Profiling and Analysis of MediaBench

 

Term Project, Winter Semester 2000

Guillaume Thomassin

Supervisors: M. Platzner & R. Enzler

 
 

Abstract

There is a great demand for more powerful handheld devices: they are asked to be cheap, have long battery life and still be performant. The ZIPPY project aims are to fulfill these goals by designing a new dynamically reconfigurable processor architecture. 

In this term project, the multimedia-oriented MediaBench benchmark suite was ported to the SimpleScalar processor simulator tool set. Appart from the PGP application, the port does not need big modifications in the MediaBench programs nor in SimpleScalar. Nine of the eleven programs of MediaBench were analyzed with the SimpleScalar simulators and with post-processing scripts. Two different processor architectures were simulated: (a) a StrongARM like architecture and (b) the default SimpleScalar architecture, which is a general purpose processor model.

The results are that the programms rely heavily on integer operations, that the branch prediction hit rates are not really high due to data dependent branches, the data cache hit rates greatly vary in function of the program. An estimation of the applications' memory footprint was calculated. The runtime intensive functions were outlined in all the applications. The different behaviour depending on the processor architecture chosen was also analyzed.
