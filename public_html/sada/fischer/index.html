Processor Wizard Visualizer

Term Project, Winter Semester 2002/03

Marco Fischer

Supervisors: Rolf Enzler & Marco Platzner

 
 

Abstract

Im Rahmen des Processor Wizard Projekts wird für eine bestimmte Applikation die optimale Prozessorarchitektur gesucht. Dies kann als Optimierungsproblem auf den Parametern der CPU aufgefasst werden. Mit einem Optimierungsalgortihmus wird dann iterativ versucht, die CPU-Architektur zu optimieren. Es handelt sich dabei um eine multikriterielle Optimierung. Die Kriterien sind Rechenleistung, Kosten (Chipfläche) und Leistungsverbrauch. Die Performance der CPU wird mit einem Prozessorsimulator ermittelt.

Das Ziel dieser Semesterarbeit war, Graphical User Interfaces für textuelle Prozessorkonfigurationen und Optimierungsverläufe zu entwickeln. Dazu sind in einem ersten Schritt geeignete GUI Toolkits evaluiert worden.

Während dieser Arbeit sind zwei Visualisierungstools entwickelt worden. Einerseits erlaubt es der Config Viewer, eine einzelne textuelle Konfigurationsdatei einer CPU-Architektur wie sie der Simulator verwendet, zu visualisieren. Dargestellt werden die drei Komponenten Branch Predictor, Processor Core und Memory Hierarchy. Parameter koennen interaktiv geaendert und wieder in Konfigurationsdateien gespeichert werden.

Auf der anderen Seite ermöglicht der Pareto Visualizer, die Resultate eines Optimierungsverlaufs darzustellen. Es koennen mehrere Iterationen der Optimierung und Pareto-Punkte (kein anderer Punkt ist in allen Kriterien 'besser') bzw. Pareto-Fronten gezeigt werden. Zahlreiche Optionen erlauben eine individuelle Darstellung der Kurven und Punkte, insbesondere für Vorführungen oder Präsentationen.

Die Erfahrungen haben gezeigt, dass es nicht ganz einfach ist, grosse Schwankungen der Wertebereiche verhältnismässig und ansprechend zu visualisieren. Mit dem Config Viewer scheint aber ein akzeptabler Kompromiss gefunden.

Beide Tools stehen als abgeschlossene und lauffähige Standalone Versionen zur Verfügung. In beiden steckt noch weiteres Entwicklungspotential. Wünschenswert wäre eine Kopplung der beiden Tools.
